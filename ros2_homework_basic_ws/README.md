3.根据网上视频的介绍，我意识到有几个概念我必须先了解一下，即：节点、话题、动作。我去看了赵虚左的介绍：
node(节点）：就有点像是每个子系统的功能主体，比如雷达/相机，可以发布信息
topic(话题）：能实现跨语言（py与cpp)通信，同样topic的node可以相互通信
通信模型：没有实战很抽象
interface（接口）：是数据载体，接口文件有三种:msg、srv、action
4.我又去看了一下鱼香ros的例程：
1）首先他介绍了一下几个话题，并且提到两个库：turtlesim（功能包）和geometry\_msgs（信息接口包）
前者是海龟客户端，我可以用内置的各种api调教这只海龟;后者是一个几何参数的定义标准
要创建一个cpp编写的功能包
2）开始编写cpp文件/py文件，rcpcpp应该是一个cpp中关于通信底层逻辑定义的api,其他的理解见文件，写的挺似懂非懂的。最后实现了，至于文件里的几个提示，回答如下：
Q:乌龟的运动消息格式是什么？A:"geometry\_msgs::msg::Twist"
Q:什么参数能控制乌龟的转向？A:msg.angular.*
Q:另外，如何保证乌龟的画两个圆是相切的？A:要通过反馈控制，感觉有点像pid,不管精度再小都需要补偿他的静态误差，因为高精度他是到不了的。或者人眼内的“相切”，那就是细微运动来逼近这个pi
3)补充：运行前要加入环境变量，这样不用到build里面。source install/setup.bash
